编码规范

  1. 环境搭建
    • 安装vss
    • 目录结构：在D盘下创建目录
      • exlipse
      • project
        • t9
    • JRE设置，采用Java6
    • Java代码统一采用UTF-8编码
    • 代码格式要求
      • 设置一下缩进 2，不使用Tab
      • 大括号采用紧缩排版样式
      • 运算符左右留空格
    • 在D:\project\t9目录下创建Java工程t9
  2. 数据库设计
    • 每个表，都设置自增字段，SEQ_ID
    • 均采用大写
    • 采用下划线做分隔符
  3. 命名规范
    • T9前缀 类
    • T9I 接口
    • 包名均采用小写，长度不大于10
  4. 需求文档规范
    • 参与者
    • 功能需求
    • 非功能需求
    • 主要业务流程（以功能模块为单位）
    • 注意事项（以功能模块为单位）
  5. 任务提交验收规范
    • 代码
    • 测试用例和测试结果
    • 使用文档
      • 应用场景介绍
      • 接口，包括配置参数接口，构造函数、方法接口
      • 典型案例，对应于使用场景
  6. 每天下班前，进行代码提交
  7. 代码备份，每天都进行
