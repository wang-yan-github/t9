代码物理目录部署及其命名

  1. 客户端
    • core
      • js //通用JS文件
      • inc //通用JSP文件
      • styles //系统样式、图片
      • org //组织单位管理
      • dept //组织机构管理
      • role //角色管理
      • msrg //消息系统
      • mail //邮件系统
      • codeclass //小编码管理
    • subsys
      • oa //OA
      • wf //工作流
      • hr //人力资源
      • crm //客户关系
      • fis //财务
      • ea //进销存
    • rad
      • dsdef //数据字典管理
    • raw //未成熟版代码
    • test //测试代码
  2. 服务器端
    • t9.core //系统核心层
      • act //活动控制层
      • data \ dto //数据结构、数据接口定义
      • logic //逻辑实现
      • exps //异常
      • global //常量定义
      • db //数据库相关
      • util //工具类
      • codeclass //小编码
    • t9.rad //快速开发
      • dsdef
    • t9.subsys //子系统
      • oa
    • raw //未成熟版代码
    • test //测试代码
  3. 开发阶段
    • 直接在各自的包中写代码，否则会带来很多麻烦
    • raw中，仅是局部代码测试用
  4. 命名
    • 业务英文描述的缩写
    • T9前缀
    • 后缀：Act \ Exception \ Const \ Utility \ Db
