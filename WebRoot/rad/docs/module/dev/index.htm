开发模式 开发步骤
1. 建立数据库表 结构 在数据库中建立起要开发的模块的数据结构
建表的规则为：
    1.表名称和字段名的命名规则为：名称大写，且单词与单词之间使用下划线连接
    2.每一张表都存在一个唯一且自增的字段SEQ_ID
2. 建立数据字典 在系统管理菜单下的数据字典，新建中可以通过抽取数据字典按钮，可以快速填写数据字典。
3. 建立模块：
    1.在src和webroot下建立相对应的目录结构
    2.src的模块下至少应包括的子包有：act、data、logic
         act：存放处理请求的方法
         data：存放实体类
         logic： 存放用于处理具体逻辑的方法
4. 创建实体类
    实力类的创建规则为：
         实体类名：
             1.在对应的表名称前加前缀T9；
             2.所有单词首字母大写，其余小写；
             3.取消所有下划线；
             如：表名称-- OA_TEST 对应的类名为 --- T9OaTest
         字段名称：
             1.数据库中对应的字段，所有单词首字母大写，其余小写；
             2.取消所有下划线；
             3.字段名称首字母小写
             如： AO_TEST_FIELD 对应类字段名为 oaTestField;
         方法：
             所有的字段都需要有get/set方法。

单表增删改查向导 单表的增删改查可以快速的搭建上述开发模式中的第3和第4步.
    向导：
        1.选择合适的模板
        2.根据数据字典建的table_no，和对应的模块名（模块位置对应的是src下的t9目录）
        3.生产代码
        4.手动调试代码的错误或不合理的地方
