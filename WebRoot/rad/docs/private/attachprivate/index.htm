附件浮动菜单权限控制 selfdefMenu 参数是附件浮动菜单用来控制权限的，下面列出来的就是各种类型的附件所对应的权限清单， 个人可以更加实际情况进行配置。
selfdefMenu:{
office:["downFile","dump","read","edit","setSign","deleteFile"], //office 文档的浮动菜单项
img:["downFile","dump",,"play","insertImg"], //图片的浮动菜单项
music:["downFile","dump","play"], //MP3等的浮动菜单项
video:["downFile","dump","play"], //视频等的浮动菜单项
others:["downFile","dump"] //其他格式文档等的浮动菜单项
}
["downFile","dump","read","edit","setSign","deleteFile","play","insertImg"] 分别对应["下载","转存","阅读","编辑","设置标引","删除","播放","图片插入到正文"]
 var selfdefMenu = {
office:["downFile","dump","read","edit","deleteFile"],
img:["downFile","dump","play","deleteFile","insertImg"],
music:["downFile","dump","play","deleteFile"],
video:["downFile","dump","play","deleteFile"],
others:["downFile","dump","deleteFile"]
}
attachMenuSelfUtil(container,moduleName,attachNames,attachIds,prec , code,seqId,selfdefMenu);
参数注释:
* @param container 附件显示的HTML容器的 ID
* @param moduleName 附件所属模块名
* @param attachNames 附件名称串，以×隔开
* @param attachIds 附件的ID串，命名规则详细参见文件上传下载
* @param prec 附件前缀，主要是为了解决同一个页面多个附件组建的问题
* @param code 附件组建后缀，作用同上
* @param seqId 传入的特殊值，文件柜中使用到
*@param selfdefMenu 各种类型附件的权限
